# Review Workflow

## When to Review
- After a GDD is complete (design-review)
- After code is written (code-review)
- After a story is implemented (story-done)
- Before phase transitions (gate-check)

## Review Process
1. Load the review skill
2. Read the artifact being reviewed
3. Check against standards
4. Return verdict: APPROVE / CONCERNS / REJECT

## Review Gate Modes
| Mode | Behavior | Best For |
|------|----------|----------|
| full | All gates active | Important decisions |
| lean | Phase gates only | Default |
| solo | No gates | Prototypes |
